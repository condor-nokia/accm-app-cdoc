= How to retrieve alarms as {vno-initials}

== Introduction

This section describes how to use the {app-short-name} to retrieve alarms as an {vno-initials}.

When {app-short-name} is installed over Altiplano, the {vno-initials} can retrieve all the alarms
through the WebUI and the application proxy(using REST queries).

When the {vno-initials} wants to execute REST queries to index search,
instead of using the Altiplano's URL:
https://{altiplano-host}/altiplano-indexsearch/
it must point to {app-short-name} proxy URL:
https://{altiplano-host}/{wnp-alarm-context}/

{app-short-name} is capable of returning the alarms that belongs to the {vno-initials} discarding
the alarm types that are set in the blocked list configured by the {inp-initials} user.
Refer to xref:User_Guide:How_to_use_WNP_as_INP.adoc[How to Use {app-short-name} as {inp-initials}] section to
know how to configure blocked alarm types globally and per {vno-initials}.

[NOTE]
====
Before using {app-short-name} as an {vno-initials} you must be authenticated as {vno-initials} user.
Refer to xref:User_Guide:How_to_manage_WNP_access.adoc[How to Manage {app-short-name} access] section to know the required steps.
====

== Retrieve active alarms
When an {vno-initials} wants to retrieve active alarms from Index Search
it must use the {app-short-name} WebUI or the {app-short-name} Index Search REST Proxy directly.

=== Using the {app-short-name} WebUI

To view the active alarms using the WebUI, execute the following steps:

. Click the "Alarms" option on the menu bar displayed on top of the screen.
+
image::vno_portal/Alarms_menu.png[align="center"]

. All Active Alarms will be listed:
+
image::vno_portal/AlarmsSection_listActiveAlarms.png[align="center"]

. Click the filter button displayed on left of the screen.
+
image::gridFilter.png[align="center"]

. The filter options are shown:
+
image::vno_portal/AlarmsSection_listActiveAlarms_search.png[align="center"]

. Enter all filters. Click the *Apply filter* button.
+
image::vno_portal/AlarmsAndMetricsSection_applyFilter.png[align="center"]

=== Using the REST Proxy
To retrieve active alarms using the {app-short-name} alarms proxy a POST request must be sent the following URL:
https://{altiplano-host}/{wnp-alarm-context}/alarms-active/_search

[NOTE]
====
The url is the same as offered by Altiplano Index Search but the difference is that instead of pointing to
altiplano-indexsearch context the URL points to {app-short-name} proxy context (*{wnp-alarm-context}*).
====

==== Example to retrieve all the alarms
If an {vno-initials} wants to retrieve all the alarms a POST request to the url:
https://{ip-host-example}/{wnp-alarm-context}/alarms-active/_search must be sent
including an empty body or with the next one:

[source, json]
----
{
   "query": {
      "match_all": {}
    }
}

field sorted by "raisedTime", a POST request must be sent to the following URL:
https://{ip-host-example}/{wnp-alarm-context}/alarms-active/_search
including the following body:

[source, json]

{
  "size": 5,
  "from": 0,
  "query": {
    "bool": {

            "should": [
              {
                "wildcard": {
                    "alarmResource.raw": {
                      "value": "*L1.P1.NOK1*"

              }
            ]
          }
        }
      ]
    }
  },
  "sort": [
    {
      "raisedTime": {
        "order": "desc"
      }
    }
  ]
}

== Retrieve historical alarms
When an {vno-initials} wants to retrieve historical alarms from Index Search
it must use the {app-short-name} WebUI or the {app-short-name} Index Search REST Proxy directly.

To view the historical alarms using the WebUI, execute the following steps:

. Click the "Alarms" option on the menu bar displayed on top of the screen and then the "HISTORIC" tab inside the page.

. All Historic Alarms will be listed:
+
image::vno_portal/AlarmsSection_listHistoricAlarms.png[align="center"]

. Click the filter button displayed on left of the screen.
+
image::gridFilter.png[align="center"]

. The filter options are shown:
+
image::vno_portal/AlarmsSection_listHistoricAlarms_search.png[align="center"]

. Enter all filters. Click the *Apply filter* button.
+
image::vno_portal/AlarmsAndMetricsSection_applyFilter.png[align="center"]

=== Using the REST Proxy
To retrieve historical alarms using the {app-short-name} alarms proxy a POST request must be sent the following URL:
https://{altiplano-host}/{wnp-alarm-context}/alarms-history-*/_search

[NOTE]
====
The url is the same as the offered by Altiplano Index Search but the difference is that instead of pointing to
altiplano-indexsearch context the URL points to {app-short-name} proxy context (*{wnp-alarm-context}*).
====
