= How to monitor campaign progress

== Introduction

This section provides the steps required to monitor the progress of the campaigns.

{app-short-name} provides a REST Interface and a WebUI for operators to monitor the campaigns.

[NOTE]
====
Before monitoring the progress, you must be authenticated to the {app-short-name} as an altiplano user.
Refer to xref:User_Guide:How_to_manage_ACCM_access.adoc[How to Manage {app-short-name} access] section for more details.
====

== Using the REST API

=== Monitor the progress of a campaign

To get the progress of a campaign using the {app-short-name} REST API you must execute the following step:

. Send a GET request to the following URL:
link:https://{altiplano-host}/{accm-api-context}/campaigns/<CAMPAIGN_NAME>/progress[https://{altiplano-host}/{accm-api-context}/campaigns/<CAMPAIGN_NAME>/progress,window=_blank]
+
Path Variable::
+
* *CAMPAIGN_NAME*: It is the string that identifies the campaign.
+
Body::
The request body must be empty.
+
For example:: A GET request must be sent to the following URL: https://{ip-host-example}/{accm-api-context}/campaigns/c1/progress

==== Response schemas

===== CampaignProgressSchema
[cols="1,1,2", options="header"]
|===
| Attribute | Type | Description
| name | string | Campaign name
| status | Status | Current status for campaign
| total-elements | int | Elements to be updated (sum of all cluster elements)
| updated-elements | int | Current elements updated (sum of all updated cluster elements)
| progress | float | Relation in percent between total and updated
| clusters | Array<ClusterProgressSchema> | List of associated clusters for the campaign and their current progress
|===

===== ClusterProgressSchema
[cols="1,1,2", options="header"]
|===
| Attribute | Type | Description
| cluster-name | string | Cluster name
| status | Status | Current status for cluster
| totalElements | int | Elements to be updated
| updatedElements | int | Current elements updated
| progress | float | Relation in percent between total and updated
| elements | Array<ClusterElementTypeProgress> | List of element types for a cluster
| executionSteps | Array<ExecutionStepSchema> | List of steps (activities on Altiplano) and their status.
|===

===== ClusterElementTypeProgress
[cols="1,1,2", options="header"]
|===
| Attribute | Type | Description
| elementType | string | Classification of the element within the cluster topology. For Leaf/Spine clusters, the element types are Spine, CP Leaves, Access Leaves, and ONTs. For Standalone clusters, the element types are Headend OLT, Remote OLT, and ONTs. The ONT element types will only be available for campaigns associated with an "ONT Only” software matrix.
| elements | Array<ClusterElementProgress> | List of elements (devices or ONTs) belonging to the cluster's elementType.
|===

===== ClusterElementProgress
[cols="1,1,2", options="header"]
|===
| Attribute | Type | Description
| name | string | Element’s name (device or ONT)
| status | Status | Status for the element (Content provided through Altiplano's statusPerDevice endpoint.)
| progress | float | Value obtained through Altiplano's statusPerDevice endpoint.
|===

===== ExecutionStepSchema
[cols="1,1,2", options="header"]
|===
| Attribute | Type | Description
| name | String | Name of the execution step
| status | Status | Current status of the execution step.
|===

===== Status (Enum)
*Type:* string

* NotScheduled
* Scheduled
* Executing
* Success
* Error
* ExecutingRollback
* RollBacked
* RollBackError
* Canceled

== Using the {app-short-name} WebUI

=== Monitor the progress of a campaign

To monitor the progress of a campaign using the {app-short-name} WebUI, you must execute the following steps:

. Click the "CAMPAIGNS" option on the menu bar displayed on top of the screen.
+
image::CAMPAIGNS_menu.png[align="center"]

. All the campaigns will be listed:
+
image::CAMPAIGNS_menuSection.png[align="center"]

. Locate the campaign to monitor, and click the Detail button on the row.
+
image::CAMPAIGNSSection_detail.png[align="center"]

. The general progress for the campaign is shown:
+
image::CAMPAIGNSSection_GeneralProgress.png[align="center"]

. Locate the cluster to monitor, and click the Details button on the row.
+
image::CAMPAIGNSSection_detail.png[align="center"]

. The general progress for the selected cluster is shown:
+
image::CAMPAIGNSSection_clusterDetails.png[align="center"]

. In case of error, the cause is shown at cluster level, and every step has its own cause:
+
image::CAMPAIGNSSection_ClusterDetailsErrors.png[align="center"]
+
image::CAMPAIGNSSection_ClusterDetailsErrors2.png[align="center"]
