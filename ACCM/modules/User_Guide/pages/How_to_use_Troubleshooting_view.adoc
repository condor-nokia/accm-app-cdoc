= How to Use Troubleshooting View

== Introduction

This section describes how to use the Troubleshooting View as an {vno-initials}.

The troubleshooting view allows an ISP user to understand the status of network elements
related to a particular customer.
It also provides relevant information for diagnosing problems or verifying specific metrics.

== Using the Troubleshooting View

To access the Troubleshooting View using the WebUI, execute the following steps:

. Click the "Troubleshooting" option on the menu bar displayed on top of the screen.
+
image::vno_portal/Troubleshooting_menu.png[align="center"]

. Click the filter select displayed on left of the screen.
+
image::vno_portal/Troubleshooting_searchCustomer.png[align="center"]

. Type the customer name to search and then select the desired customer.
+
image::vno_portal/Troubleshooting_typeCustomer.png[align="center"]

. Information similar to the following will be displayed:
+
image:vno_portal/Troubleshooting_initialInfo.png[align="center"]
+

Please refer to the next section to check the information details.

== Information details

In this section, all the information available at the Troubleshooting view is explained.

Once the desired customer was selected, a breadcrumb menu appears on top. This menu
contains the fiber, the ont and the services of the customer.
Each element has an icon to its right that represents the intent health status.

The information shown when clicking each element is explained below.

=== Fiber Info

When the fiber is selected the next information is shown:

image:vno_portal/Troubleshooting_headerInfo_fiber.png[align="center"]

* Target: It is the target of the intent selected.
* Required Network State: It represents the selected intent's state in the network.
* Alignment State: It is the alignment state of the selected intent. The alignment state can be: "Aligned" or "Misaligned".
* Last Available Health: It is the last health state of the selected intent. The health state can be: "Healthy", "Unknown", "At-risk", "Faulty", "Suspended" or "No data".

[IMPORTANT]
====
The fiber item can be available or disabled depending on the readable intent type list configured
when installing the application.
Refer to xref:Installation-and-Deployment-Guide:How_to_install_and_configure_WNP_application.adoc#_config[{app-short-name} Config] section for more details.
====

=== {ont-initials} Info

When the {ont-initials} is selected the next information is shown:

image:vno_portal/Troubleshooting_headerInfo_ont.png[align="center"]

* The header that contains the following information:
** Target: It is the target of the intent selected.
** Required Network State: It represents the selected intent's state in the network.
** Alignment State: It is the alignment state of the selected intent. The alignment state can be: "Aligned" or "Misaligned".
** Last Available Health: It is the last health state of the selected intent. The health state can be: "Healthy", "Unknown", "At-risk", "Faulty", "Suspended" or "No data".
* Active Alarms: The active alarms of the selected intent.
* Historic Alarms: The historic alarms of the selected intent.
* Details: It is the information retrieved when executing the {main-base-app}'s action called "Show ONT Info(module: ont-action, action: show-ont-info)" over the {ont-initials}.

[IMPORTANT]
====
The "Details" tab will be only present if Altiplano's support the "ONT Info" action (Added in Altiplano 24.9).
====


=== UNI Info

The information that appear in the breadcrumb is composed by "UNI - Service Name".

When the UNI/service is selected the next information is shown:

* The header that contains the following information:
** Target: It is the target of the intent selected.
** Required Network State: It represents the selected intent's state in the network.
** Alignment State: It is the alignment state of the selected intent. The alignment state can be: "Aligned" or "Misaligned".
** Last Available Health: It is the last health state of the selected intent. The health state can be: "Healthy", "Unknown", "At-risk", "Faulty", "Suspended" or "No data".
* Active Alarms: The active alarms of the selected intent.
* Historic Alarms: The historic alarms of the selected intent.
* Details: It is the information retrieved when executing the {main-base-app}'s action called "Show UNI Details(module: ont-uni-action, action: show-uni-detail)" over the UNI.
* DHCP Sessions: It is the information retrieved when executing the {main-base-app}'s action called "Show DHCP Sessions(module: anti-spoofing-action, action: show-dhcp-sessions)" over the UNI.
* VLAN Sub-Interface Information: It is the information retrieved when executing the {main-base-app}'s action called "Show VSI Info(module: show-vsi-info, action: show-vsi-info)" over the UNI.
* FDB MAC Address: It is the information retrieved when executing the {main-base-app}'s action called "Show FDB MAC(module: show-fdb-mac, action: show-fdb-mac)" over the UNI.

[IMPORTANT]
====
The tabs related to actions will be only present if Altiplano's support them(Added in Altiplano 24.9).
====

== Actions

The actions that can be executed are found on top right.

image::vno_portal/TroubleshootingSection_LiveCollections.png[align="center"]

=== Start live monitoring

Depending of the selected element on the breadcrumb: the fiber, the ont or the services of the customer, a live collection can be
executed over the element.

. Click the *Start live monitoring* icon on top right.
+
image::vno_portal/TroubleshootingSection_LiveCollectionsBtn.png[align="center"]

. A popup with the live collection info to start is opened:
+
image::vno_portal/TroubleshootingSection_startONTLiveCollection.png[align="center"]

. Click the *START* button.

=== Reaload

Depending of the selected element on the breadcrumb: the fiber, the ont or the services of the customer, the information about the element can be reloaded.

. Click the *Reload* icon on top right.
+
image::vno_portal/TroubleshootingSection_reloadBtn.png[align="center"]

The header information, Active Alarms, Historic Alarms and Details of the element will be realoaded.
