== Using the REST API

=== Manual start
To manually start a campaign using the {app-short-name} REST API, you must execute the following step:

. Send a POST request to the following URL:
link:https://{altiplano-host}/{accm-api-context}/campaign-manager-api/campaigns/<CAMPAIGN_NAME>/manual-start[https://{altiplano-host}/{accm-api-context}/campaign-manager-api/campaigns/<CAMPAIGN_NAME>/manual-start,window=_blank]
+
Path Variable::
+
* *CAMPAIGN_NAME*: It is the string that identifies the campaign.
+
Body::
+
The request must contain the following JSON body:
+
For example:: A POST request must be sent to the following URL: https://{ip-host-example}/{accm-api-context}/campaign-manager-api/campaigns/campaign1/manual-start

[NOTE]
====
The access token of the user must be sent in the request as a Bearer Authorization HTTP Header.
To retrieve a token, follow the steps in the xref:How_to_manage_ACCM_access.adoc[REST APIs] section.
====

=== Rollback
To rollback a campaign using the {app-short-name} REST API, you must execute the following step:

. Send a POST request to the following URL:
link:https://{altiplano-host}/{accm-api-context}/campaign-manager-api/campaigns/<CAMPAIGN_NAME>/clusters/<CLUSTER_NAME>/elements/<ELEMENT_NAME>/rollback[https://{altiplano-host}/{accm-api-context}/campaign-manager-api/campaigns/<CAMPAIGN_NAME>/clusters/<CLUSTER_NAME>/elements/<ELEMENT_NAME>/rollback,window=_blank]
+
Path Variable::
+
* *CAMPAIGN_NAME*: It is the string that identifies the campaign.
* *CLUSTER_NAME*: It is the string that identifies the cluster.
* *ELEMENT_NAME*: It is the string that identifies the element to be rolled back.
+
Body::
+
The request must contain the following JSON body:
+
For example:: A POST request must be sent to the following URL: https://{ip-host-example}/{accm-api-context}/campaign-manager-api/campaigns/campaign1/clusters/LSN/elements/Condor_Spine1/rollback

[NOTE]
====
The access token of the user must be sent in the request as a Bearer Authorization HTTP Header.
To retrieve a token, follow the steps in the xref:How_to_manage_ACCM_access.adoc[REST APIs] section.
====

== Using the {app-short-name} WebUI

=== Manual start
To manually start a campaign using the {app-short-name} WebUI, you must execute the following steps:

. Click the "CAMPAIGNS" option on the menu bar displayed on top of the screen:
+
image::CAMPAIGNS_menu.png[align="center"]

. All the campaigns will be listed:
+
image::CAMPAIGNS_menuSection.png[align="center"]

. Locate the campaign to start and click the Manual Start button on the row. Please note that the Manual Start option is only available for NotScheduled campaigns:
+
image::CAMPAIGNSSection_notScheduled.png[align="center"]

. Confirm the action:
+
image::CAMPAIGNSSection_confirmManualStartAction.png[align="center"]

=== Cancel
